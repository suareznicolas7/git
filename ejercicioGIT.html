Examen Git y Github

Alumn@: [NSP]

  1. ¿Que es un sistema de control de versiones?

    Herramienta que ayuda a los equipos de desarrollo de software a gestionar los cambios en el código fuente.

  2. ¿Cual es la diferencia entre git y github?

    Git es un sistema de software local para controlar las versiones y el historial de cambios en el código, mientras que GitHub es una plataforma en la nube (sitio web) que aloja repositorios de Git para facilitar el trabajo colaborativo.

  3. ¿Cómo creo una rama en git?

    git-branch-nombre_rama

  4. ¿Cómo clono un repositorio?

    git clone y el enlace del ssh.

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Con git pull.

  6. ¿Cómo puedo generar un tag?

    git tag nombre_tag

  7. Cómo hago una mezcla de ramas?

    git checkout branch_b git merge branch_a

  8. ¿Cómo puedo moverme a un commit especifico?

    git checkout

  9. ¿Cómo puedo borrar una rama?

    git branch -d nombre_rama

  10. ¿Cómo puedo borrar un tag?

    git tag -d nombre_tag

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    git status

  12. ¿Para que sirve el comando git remote -v?

    Lista los repositorios remotos configurados en tu proyecto local.

  13. ¿Cómo le digo a git cual es mi email y correo?

    git config --global user.name "Tu Nombre" git config --global user.email "tu@email.com"

  14. ¿Cuales son los pasos para conectar git con github?

    git remote add origin. Los comandos principales son git config, git init, git add, git commit, git remote add y git push.

  15. ¿Cómo puedo crear un alias en git?

    Para crear un alias en git y simplificar comandos largos, usa git config --global alias. alias_nombre comando_original.
